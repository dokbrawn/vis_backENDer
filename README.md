# Backend сервер для сбора и визуализации данных о качестве сотовой связи

Настольное приложение на C++17, выполняющее приём данных с Android-клиента через ZeroMQ, запись в PostgreSQL, отображение в реальном времени с помощью ImGui/ImPlot и построение тепловых карт на тайлах OpenStreetMap методом IDW.

Является серверной частью расчётно-графической работы «Программно-аппаратный комплекс для сбора и визуализации данных о качестве сотовой связи (GSM/LTE/5G)».

## Возможности

- Приём JSON-пакетов от Android-приложения по протоколу ZeroMQ (паттерн PUSH/PULL).
- Парсинг данных: GPS-координаты, параметры сот (RSRP, RSRQ, RSSI), статистика сетевого трафика.
- Сохранение измерений в базу данных PostgreSQL (таблицы `user_equipment`, `location_history`, `telephony_history` и др.).
- Графический интерфейс на Dear ImGui с несколькими вкладками:
  - «Главная» — статус подключения, количество принятых пакетов.
  - «Графики» — временные ряды RSRP, RSRQ, RSSI (ImPlot).
  - «Локальные» — таблица последних измерений.
  - «Карта» — отображение тайлов OpenStreetMap с наложением тепловой карты сигнала.
  - «Телефония» — детальная информация о сотах.
  - «Фильтры» — отправка команд на Android для включения/выключения сбора GPS, LTE, NR.
- Асинхронная загрузка и кэширование тайлов OSM (проекция Web Mercator, через cURL).
- Генерация тепловой карты уровня сигнала (RSRP) методом интерполяции обратно взвешенных расстояний (IDW) с ограничением радиуса.
- Многопоточность: отдельный поток для ZMQ-приёмника, основной поток для GUI; синхронизация через `std::mutex` и `std::atomic`.

## Зависимости

- **C++17** (поддерживаемые компиляторы: GCC 9+, Clang 10+, MSVC 2019+)
- [ZeroMQ](https://zeromq.org/) (libzmq, cppzmq)
- [nlohmann/json](https://github.com/nlohmann/json) (парсинг JSON)
- [libpq](https://www.postgresql.org/docs/current/libpq.html) (PostgreSQL C API)
- [Dear ImGui](https://github.com/ocornut/imgui) + [ImPlot](https://github.com/epezent/implot)
- [GLFW](https://www.glfw.org/) (оконный менеджер для ImGui)
- [libcurl](https://curl.se/libcurl/) (загрузка тайлов OSM)
- [stb_image](https://github.com/nothings/stb) (загрузка PNG для тайлов)
- OpenGL (для ImGui)

Сборка осуществляется с помощью CMake.

## Сборка и запуск

### 1. Установка зависимостей

На Ubuntu/Debian:
```bash
sudo apt install build-essential cmake libzmq3-dev libpq-dev libcurl4-openssl-dev libglfw3-dev libglew-dev
```

Дополнительно потребуется установить заголовки ImGui и ImPlot (можно скопировать в папку `external/` или подключить через CMake `FetchContent`).

### 2. Клонирование репозитория

```bash
git clone <url-репозитория>
cd rework_osm_step14
```

### 3. Настройка CMake и компиляция

```bash
mkdir build && cd build
cmake ..
make -j$(nproc)
```

### 4. Настройка PostgreSQL

Создайте базу данных (например, `backend_control`) и таблицы, используя SQL-скрипт из `scripts/init_db.sql` (при наличии) или выполнив следующие команды:

```sql
CREATE DATABASE backend_control;
\c backend_control
-- далее DDL для таблиц user_equipment, location_history и др.
```

Убедитесь, что параметры подключения (хост, порт, имя БД, пользователь, пароль) прописаны в файле `config.json` или заданы через переменные окружения/аргументы командной строки.

### 5. Запуск

```bash
./build/backend_server
```

Приложение откроет GUI-окно и начнёт прослушивать ZMQ-порты:
- **5558** — приём данных от Android-клиента.
- **5559** — приём команд (отправка фильтров обратно на клиент).

## Использование

1. Запустите сервер.
2. На Android-устройстве запустите приложение `visprog-feature-geolocator`, предварительно указав IP-адрес сервера в настройках.
3. На вкладке «Главная» появится счётчик принятых пакетов и данные о местоположении.
4. Вкладка «Графики» покажет динамику изменения сигнала в реальном времени.
5. Вкладка «Карта» загрузит тайлы OpenStreetMap и по мере накопления точек начнёт строить тепловую карту. Радиус интерполяции и степень (power) можно изменять в интерфейсе.
6. Вкладка «Фильтры» позволяет включать/выключать сбор отдельных типов данных на клиенте (GPS, LTE, NR).
7. Данные автоматически сохраняются в PostgreSQL для дальнейшего анализа.

## Ключевые модули

- **`RuntimeState`** – потокобезопасное хранилище всех текущих данных (местоположение, список сот, точки для тепловой карты). Доступ защищён `std::mutex`.
- **`PgSqlMinimal`** – инкапсулирует подключение к PostgreSQL через libpq, выполняет параметризованные вставки в таблицу `user_equipment` и вспомогательные таблицы.
- **`ZmqReceiver`** – выделенный поток, принимающий JSON‑сообщения с порта 5558, разбирающий их с помощью `nlohmann/json` и обновляющий `RuntimeState`.
- **`GuiApp`** – главный цикл приложения на ImGui + ImPlot, содержащий логику всех вкладок и отрисовку карты.
- **`OsmTileFetcher`** – асинхронный загрузчик тайлов OpenStreetMap с кэшированием на диске и декодированием PNG.
- **`HeatmapGenerator`** – строит тепловую карту методом IDW по накопленным точкам, учитывая радиус в метрах и геодезическое расстояние (формула гаверсинуса). Результат – RGBA‑текстура, накладываемая на карту.

## Команда для Android-клиента

Взаимодействие осуществляется через два ZMQ‑сокета:
- **Данные** (порт 5558) – Android отправляет PUSH‑сообщения, сервер принимает PULL.
- **Команды** (порт 5559) – сервер отправляет PUSH‑сообщения, Android принимает PULL. Формат команд: `{"gps": true/false, "lte": true/false, "nr": true/false}`.

## Примечания

- Тепловая карта ограничена 10 000 точками; старые точки автоматически удаляются.
- Для работы карты требуется предварительная загрузка тайлов OSM; возможно, потребуется активное интернет-соединение при первом запуске.
- Программа разработана и протестирована на Windows 11.
